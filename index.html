// FAULT ON THE FLOOR

One model for everything is not a strategy.

Defaulting every task to a single frontier lane wastes judgment where you need force, and force where you need judgment. No deliberate advisor / executor split means every child inherits the same blind spots.

  • No named loadouts — just ad-hoc flags and hope
  • Advisor and executor fused into one opaque session
  • Fleets improvised without contracts or RESULT lines

// ORDNANCE ON RACK

Arm the right loadout. Keep the advisor sharp.

Named loadouts

Each bay is a known lane — model, effort, rules, worktree discipline. Pull grok-high when the brief needs heavy lifting.

Fable arms executors

Claude Code stays the advisor. On explicit instruction it arms Grok 4.5 children via the armory launcher — deliberate hybrid, not auto-dispatch.

Fleet of children

Manifest in, parallel worktrees out. One commit per task, PROGRESS.md ledger, single RESULT line.

// QUICKSTART

Install. List. Arm. Fleet.

Click any line to copy. npm is the default path — same as SAGE.

setup
$ npm install -g llm-armory ⎘
#1 Install globally — bins: armory · llm-armory · llm
$ armory --list ⎘
#2 Inspect the arsenal rack
$ armory doctor ⎘
#3 Check PATH, providers, loadouts
$ armory grok-high -p "your brief" -w my-exec ⎘
#4 Arm a Grok 4.5 executor in a worktree
$ npx llm-armory --list ⎘
#1 Run without a global install
$ npx llm-armory doctor ⎘
#2 Health checklist
$ npx llm-armory grok-high -p "your brief" -w my-exec ⎘
#3 Arm on demand
$ git clone https://github.com/muslewski/llm-armory.git ⎘
#1 Clone the product repo
$ cd llm-armory && npm install && npm link ⎘
#2 Link armory from source
$ armory --list ⎘
#3 Same rack, local checkout
✓ Fleet: armory fleet grok-high --manifest fleet.txt
